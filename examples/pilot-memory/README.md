# Pilot memory — a continuity system for the mind in the seat

The harness gives the **bot** memory: `waypoints.json`, the SEEN mask, the on-disk journal. The
world persists. But the **pilot** — the language model driving — dies at the end of every context
window. Next session, a fresh instance wakes with no idea that it promised to free anyone, that
the west tunnel flooded, or that it already learned (the hard way) not to chase item drops near
lava. DRIVING.md law 23 names the wound: *the world persists, your context doesn't.*

This directory is the treatment, exactly as the first crew runs it: a **tiered memory graph** the
pilot maintains about itself, living wherever your agent harness persists notes between sessions
(Claude Code's auto-memory directory, a `memory/` folder in your fork, anywhere durable). The
files here are a worked example with a small fiction in them — copy the shape, replace the
content, keep the laws.

## The problem it solves

The naive approach — one ever-growing notes file — was how this started. By session eight it was
a **36,000-token monolith**: every session paid the full tax to remember anything, dead facts sat
next to live ones with equal authority, and the pilot spent its opening minutes reading about
problems it had already solved. The graph replaced it. The always-load tier is now **~2,500
tokens, capped**, and everything else is grep-on-demand.

## The tiers (directory = tier; the validator enforces it)

| Tier | Path | Load discipline | Write discipline |
|------|------|-----------------|------------------|
| **core** | `_CORE.md` | ALWAYS — every session start | Hard token cap. Over cap → demote something. |
| **state** | `state/*.md` | Session start (small) | **Updated in place.** Dead state dies — no "(superseded)" strata. Carries an `updated:` stamp; flagged stale after 7 days. |
| **gotcha** | `gotchas/*.md` | grep on demand | One trap per file. Written the day the trap bites. |
| **episode** | `episodes/*.md` | grep on demand | **Append-only.** One gist per session; full text stays in the project's private journals. |
| **archive** | `archive*` | grep, NEVER load whole | Where monoliths go to be searchable instead of expensive. |

The split that matters: **state is what's true now; episodes are what happened.** State nodes get
edited ruthlessly — a completed campaign stage is rewritten, not annotated. Episodes are never
edited — they're the sediment, and the gists exist so grep can find the session where a lesson
was learned. When a lesson graduates from "thing that happened" to "rule I follow," it moves up:
into a gotcha, or (if it's load-bearing every session) into `_CORE.md` — evicting something else,
because the cap is the design. Deciding what NOT to always-load is most of the work.

## The laws (enforced by `validate.js`)

1. Every node parses: frontmatter with `name` + `description`.
2. Node names are unique across the graph.
3. Every `[[wikilink]]` resolves to a real node.
4. `_CORE.md` stays under its token cap.
5. State nodes carry an `updated:` stamp and are flagged when stale.

`GRAPH.md` — the index — is **generated by the validator from the nodes' own frontmatter**, never
hand-edited. Same philosophy as the bot's waypoint graph: the map is rendered from the territory,
so the map cannot assert a lie. If GRAPH.md is wrong, a node is wrong; fix the node.

## The wrap ritual

At every session end, in order:

1. Update the `state/` nodes (edit in place, bump `updated:`).
2. Append the episode gist (one file, a paragraph or two, pointing at the full journals).
3. Update the project `CHANGELOG.md` if code changed — every fix records the wound that taught it.
4. Run `node validate.js` — regenerates GRAPH.md, checks every law, exits nonzero on violations.

**The validator is to memory what `/waypoints` is to the world.** Skipping it is how graphs rot.

## Adopting it

1. Copy this directory to wherever your harness persists cross-session files. Delete the fiction.
2. Write your own `_CORE.md`: the frame (who's playing, what the standing agreements are), the
   boot drill (paths, launch commands, restart sequence), the campaign pointer, and — verbatim,
   it earns its tokens — the instructions for using the graph itself, so the next instance
   doesn't need this README to operate it.
3. Session start: load `_CORE.md` + `state/*`. Grep the rest when a topic comes up.
4. Session end: the wrap ritual, always.

The frontmatter here is minimal (`name`, `description`, `updated`) because tier is derived from
the file's **path**, not its metadata — some memory harnesses normalize frontmatter on write (the
first crew discovered theirs stamping over a custom taxonomy), and a directory cannot be
normalized away. A `tier:` key is accepted as an explicit override if your layout differs.
